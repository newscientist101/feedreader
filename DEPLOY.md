# Deploying feedreader with Docker Compose

This guide walks through deploying feedreader using Docker Compose with Caddy
as a reverse proxy and Authelia for WebAuthn/passkey authentication. When
you're done, your feed reader will be available at `https://your-domain/` with
passwordless passkey login.

---

## 1. Prerequisites

- A Linux VM or server with **Docker** and **Docker Compose** installed.
  - Install guide: <https://docs.docker.com/engine/install/>
  - Compose is bundled with Docker Desktop and recent Docker Engine versions.
    Run `docker compose version` to verify.
- A **domain name** pointed at your server's IP address (an `A` record, e.g.
  `feeds.example.com → 1.2.3.4`). Caddy needs this to provision a TLS
  certificate automatically via Let's Encrypt.
- Ports **80** and **443** open in your firewall (both TCP; 443 UDP is needed
  for HTTP/3 but optional).

---

## 2. Quick Start

```bash
git clone https://github.com/newscientist101/feedreader.git
cd feedreader/deploy

# 1. Set your domain
cp .env.example .env
# Edit .env — set DOMAIN=feeds.yourdomain.com

# 2. Create your user account
#    (see Section 3 below for how to generate the password hash)
$EDITOR authelia/users_database.yml

# 3. Generate secrets in authelia/configuration.yml
#    (see Section 3 below for details)
$EDITOR authelia/configuration.yml

# 4. Start
docker compose up -d
```

Then open `https://your-domain/` in a browser — you will be redirected to the
Authelia login page.

## 3. Configuration

### Setting your domain

Edit `deploy/.env` and set `DOMAIN` to your domain name:

```
DOMAIN=feeds.yourdomain.com
```

Caddy reads this at startup and provisions a Let's Encrypt certificate
automatically. DNS must resolve before you start the stack.

### Creating your user account

Edit `deploy/authelia/users_database.yml`. You need to:

1. **Choose a username** — replace `admin` with your preferred username.
2. **Set your email address** — replace `<CHANGE_ME_EMAIL>`.
3. **Generate a password hash** — run this command and paste the output into
   the `password` field:

   ```bash
   docker run --rm authelia/authelia:latest \
     authelia crypto hash generate argon2 --password 'your-password'
   ```

   The password is only used for the very first login before you register a
   passkey. After passkey registration, logins are passwordless.

Example result:

```yaml
users:
  alice:
    displayname: 'Alice'
    password: '$argon2id$v=19$m=65536,t=3,p=4$...'
    email: 'alice@example.com'
    groups: []
```

### Generating secrets

`deploy/authelia/configuration.yml` has three `<CHANGE_ME>` placeholders.
Replace each with a unique random secret generated by:

```bash
openssl rand -hex 32
```

| Placeholder | Field | Purpose |
|---|---|---|
| `<CHANGE_ME_SESSION_SECRET>` | `session.secret` | Encrypts browser session cookies |
| `<CHANGE_ME_STORAGE_ENCRYPTION_KEY>` | `storage.encryption_key` | Encrypts Authelia's SQLite database |
| `<CHANGE_ME_JWT_SECRET>` | `identity_validation.reset_password.jwt_secret` | Signs passkey registration links |

Also replace the two `<CHANGE_ME_DOMAIN>` values in the `session.cookies`
section with your domain (e.g. `feeds.example.com`):

```yaml
session:
  secret: 'aaaaaaaaaa...'   # output of openssl rand -hex 32
  cookies:
    - domain: 'feeds.example.com'
      authelia_url: 'https://feeds.example.com/authelia'
```

### feedreader config

In `deploy/docker-compose.yml`, feedreader is already wired to use Authelia:

```yaml
environment:
  AUTH_PROVIDER: authelia
```

This tells feedreader to read the `Remote-User` and `Remote-Email` headers
that Caddy copies from Authelia's auth response.

For advanced configuration (custom port, newsletter webhook, SMTP ingestion),
you can mount a `config.toml` into the container:

```yaml
feedreader:
  volumes:
    - feedreader_data:/data
    - ./feedreader-config.toml:/data/config.toml:ro
```

Run `feedreader init` locally to generate a starter config, or see
`config.example.toml` in the repository root.

## 4. First Login and Passkey Registration

1. Navigate to `https://your-domain/` in your browser.
2. You will be redirected to the Authelia login page at
   `https://your-domain/authelia/`.
3. Enter the **username** and **password** you set in `users_database.yml`.
4. After logging in, click your username in the top-right corner to open
   account settings, or go to `https://your-domain/authelia/`.
5. Click **"Register WebAuthn device"** to set up a passkey (fingerprint
   sensor, Face ID, security key, or Windows Hello).

**Finding verification links (filesystem notifier):**

Authelia is configured to write notification emails to a file instead of
sending real emails. If Authelia asks you to verify an action, check
`notifications.txt` inside the `authelia_data` Docker volume:

```bash
docker compose exec authelia cat /data/notifications.txt
```

Copy the verification link from the file and open it in your browser.

After your passkey is registered, future logins are passwordless — just tap
your fingerprint or security key.

## 5. Updating

Pull the latest images and restart the stack:

```bash
cd deploy
docker compose pull
docker compose up -d
```

Feedreader applies database migrations automatically on startup.

## 6. Backup and Restore

All persistent data lives in Docker named volumes:

| Volume | Contents |
|---|---|
| `feedreader_data` | feedreader SQLite database (`db.sqlite3`) |
| `authelia_data` | Authelia database + passkey credentials + notifications |
| `caddy_data` | TLS certificates from Let's Encrypt |
| `caddy_config` | Caddy runtime configuration |

### Backing up feedreader data

```bash
# Option 1: copy the database file out of the running container
docker compose exec feedreader cp /data/db.sqlite3 /data/backup.sqlite3
docker cp $(docker compose ps -q feedreader):/data/backup.sqlite3 ./feedreader-backup.sqlite3

# Option 2: archive the entire volume (works while the stack is stopped)
docker run --rm \
  -v feedreader_data:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/feedreader-backup.tar.gz /data
```

### Restoring

```bash
# Stop the stack first
docker compose down

# Restore from archive
docker run --rm \
  -v feedreader_data:/data \
  -v $(pwd):/backup \
  alpine sh -c 'cd / && tar xzf /backup/feedreader-backup.tar.gz'

# Restart
docker compose up -d
```

## 7. Troubleshooting

**Certificate not provisioning**

- Ensure ports 80 and 443 are open in your firewall.
- Check that your domain's DNS A record points to this server's IP.
- View Caddy logs: `docker compose logs caddy`

**Passkey not working**

- WebAuthn requires HTTPS. If you see a browser error about "not a secure
  context", the TLS certificate may not yet be provisioned — check Caddy logs.
- Passkeys are device-specific. If you switch devices, register a new passkey
  from the Authelia settings page.

**Cannot log in / login page loops**

- Check Authelia logs: `docker compose logs authelia`
- Verify `users_database.yml` has valid YAML syntax and a correct password hash.
- Make sure all three `<CHANGE_ME>` secrets in `configuration.yml` are replaced.
- Verify the `session.cookies[].domain` and `authelia_url` match your domain.

**feedreader not loading after login**

- Check feedreader logs: `docker compose logs feedreader`
- Confirm `AUTH_PROVIDER: authelia` is set in `docker-compose.yml`.
- Verify the `forward_auth` block in `Caddyfile` is copying `Remote-User` and
  `Remote-Email` headers.
